1. Arquitectura global del sitio

Vista de primer nivel: dos portales diferenciados tras autenticación, más capa de servicios y administración técnica.

🔐 Acceso
  • Inicio de sesión — /login/
    • Redirección automática según rol (ver sección 4)
  • Cierre de sesión — /logout/
🏛 Portal institucional
  • Dashboard, bandeja, expedientes
  • Conocimiento, mapa, proceso, metodología
  • Motor analógico, dictámenes, SNI, reportes
  • Roles: admin, analista, investigador, dependencia, auditor
🏛 Ventanilla ciudadana Edomex
  • Portal de reportes, registro por eje
  • Ruta gráfica de atención
  • Detalle con chat y seguimiento
  • Rol exclusivo: publico (usuario ciudadano)
⚙️ Capa de servicios
  • API REST institucional — /api/ (expedientes, MITA, geoespacial, reportes, búsqueda)
  • API ciudadanía — /ciudadania/api/ (asistente IA, centros salud, chat, ruta)
  • Admin Django — /admin/ (solo administrador)

6. Principios de diseño de navegación

  • Separación de portales: el rol publico nunca ve el menú institucional; los demás roles no ven la ventanilla ciudadana.
  • Profundidad máxima 3 niveles en la mayoría de flujos: menú → listado → detalle/acción.
  • Coherencia workflow: bandeja y expediente comparten la misma secuencia de 7 pasos con responsables claros.
  • Acciones contextuales: guardar, avanzar y dictaminar solo aparecen en el paso activo del expediente.
  • API paralela: las mismas capacidades institucionales están expuestas vía REST para integraciones futuras.
  • Redirección inteligente: tras login, cada rol aterriza en la página más relevante para su tarea principal.
