2. Descripción general

Sistema web de gestión del conocimiento y buen gobierno que integra bases interdisciplinarias, análisis geoespacial, motor analógico de evaluación, perfiles del Sistema Nacional de Investigadores (SNI), análisis legislativo y emisión de dictámenes oficiales en un flujo operativo de 7 pasos con trazabilidad y control por roles.

Objetivos técnicos

  • Centralizar información validada de múltiples disciplinas y fuentes oficiales.
  • Orquestar el proceso MITA mediante expedientes, tareas y transiciones auditables.
  • Generar evaluaciones analógicas y dictámenes de validación en PDF.
  • Exponer una API REST para integración con otros sistemas gubernamentales.

12. Comandos para correr el sistema

12.1 Instalación inicial (desarrollo)

cd "c:\Users\rocio\OneDrive\Documents\GitHub\MI"

# Crear y activar entorno virtual (Windows)
python -m venv venv
venv\Scripts\activate

# Instalar dependencias
pip install -r requirements.txt

# Aplicar migraciones
python manage.py migrate

# Cargar datos semilla, usuarios demo y proyecto simulado
python manage.py seed_mita

# Verificar configuración
python manage.py check

12.2 Arrancar el servidor de desarrollo

# Puerto por defecto (8000)
python manage.py runserver

# Puerto alternativo (si 8000 está ocupado)
python manage.py runserver 127.0.0.1:8888

URLs de acceso:

  • Plataforma: http://127.0.0.1:8000/
  • Login: http://127.0.0.1:8000/login/
  • Admin Django: http://127.0.0.1:8000/admin/
  • API health: http://127.0.0.1:8000/api/health/
Si usa el puerto 8888, sustituya 8000 por 8888 en las URLs anteriores.

12.3 Comandos de mantenimiento

# Recargar datos demo (usuarios + expedientes simulados)
python manage.py seed_mita

# Crear superusuario adicional
python manage.py createsuperuser

# Ejecutar pruebas
python manage.py test

# Recolectar archivos estáticos (producción)
python manage.py collectstatic --noinput

# Regenerar manual de usuario con pantallas
python scripts/capturar_pantallas_manual.py
python scripts/generar_manual_docx.py

12.4 Despliegue en producción

# Windows (cmd)
set DJANGO_DEBUG=False
set DJANGO_SECRET_KEY=clave-segura-generada
set DATABASE_URL=postgres://usuario:clave@host:5432/mita
set DJANGO_ALLOWED_HOSTS=midominio.gob.mx

python manage.py migrate
python manage.py collectstatic --noinput
python manage.py seed_mita

gunicorn mita_platform.wsgi:application --bind 0.0.0.0:8000
En producción use una clave secreta segura, DEBUG=False, PostgreSQL y un servidor WSGI (Gunicorn, uWSGI) detrás de un proxy inverso (Nginx/Apache) con HTTPS.

12.5 Instalación en Linux / macOS

cd MI
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_mita
python manage.py runserver 0.0.0.0:8000
